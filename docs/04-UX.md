# 04 — UX

## Design principles

1. **Calm by default.** Wallets are anxious places. Neutral palette, no flashing alerts unless there's actual money at risk. No celebrations on send (a wire transfer doesn't confetti).
2. **One job per screen.** Dashboard shows. Send sends. Bridge bridges. No combined "mega-screens" with 12 affordances.
3. **Show the irreversibility.** Every send and bridge action shows: "This cannot be undone." Once.
4. **Slow down at the dangerous parts.** Password fields, recovery export, wallet wipe — all require a deliberate gesture (typed phrase or held button).
5. **No dark patterns.** No "skip backup" hidden in tiny text. No pre-checked optional analytics. No "are you sure you want to lose money" double-negatives.
6. **Mobile-first layout.** Most retail users are on phones. Desktop is just a wider mobile view, not a different app.

## Information architecture

```
/                          → Splash + Connect (decide create/restore/unlock)
/onboarding/create         → Generate mnemonic, backup, verify, set password
/onboarding/restore        → Enter mnemonic, set password
/unlock                    → Password entry for existing wallet
/dashboard                 → Default after unlock
/send/prl                  → Send PRL flow
/send/wprl                 → Send WPRL flow
/receive                   → Address + QR (toggle PRL / WPRL)
/bridge                    → Bridge entry (toggle direction)
/bridge/status/:id         → Live tracker for a specific bridge action
/history                   → Tx history (filterable PRL / WPRL / Bridge)
/history/:txid             → Tx detail
/settings                  → Account + security
/settings/network          → Mainnet / testnet toggle
/about                     → How this works + version + build hash
```

## Screen-by-screen

### Splash (`/`)
- Logo + "Pearl Web Wallet."
- One subhead: "Non-custodial. PRL and WPRL in one place."
- Three CTAs:
  - **Create a new wallet** (primary)
  - **Restore from recovery phrase** (secondary)
  - **Unlock existing wallet** (tertiary, only shown if a wallet exists in this browser)
- Footer: "v1.0.0 · build abc1234 · github link · audit report PDF"

### Create flow (`/onboarding/create`)

Step 1 — **Generate** (~3 sec):
- Headline: "Your new wallet is ready."
- "Pick a recovery phrase length:" radio: 12 / 24 words. Default 12.
- Show the mnemonic in a clean monospace grid (3 cols × 4 rows for 12, 3 × 8 for 24). Each word numbered.
- One button: "I've written it down" — disabled for 5 seconds (forces the user to actually look).
- Tertiary action: "Copy to clipboard" — explicitly discouraged with text: "Writing down is safer. Clipboard can be read by malware."

Step 2 — **Verify**:
- "To confirm you have it, type words **3, 7, and 11** below."
- Three inputs.
- "Back" returns to step 1.
- If wrong: "Words don't match. Check what you wrote down."
- Three wrong attempts: prompt to go back and re-record.

Step 3 — **Set password**:
- "Set an unlock password. This password protects your wallet on this device."
- Two inputs (password, confirm). Strength meter (zxcvbn). Min 10 chars.
- "This password is separate from your recovery phrase. If you forget it, restore from your recovery phrase. If you lose both, your funds are gone."
- Big checkbox: "I understand there is no recovery." Required.
- "Create wallet" button.

Step 4 — **Done**:
- "Wallet created."
- "Your Pearl address: prl1p... (Tap to view full)"
- "Your Ethereum address (for WPRL): 0x... (Tap to view full)"
- Single CTA: "Open dashboard."

### Restore flow (`/onboarding/restore`)
- Heading: "Restore your wallet."
- 12 or 24 input fields (toggle). Each field has autocomplete=off, autocapitalize=off, type=password (masked by default with show-toggle).
- BIP-39 wordlist autocomplete dropdown per field (helps catch typos).
- "Restore" — disabled until all words present and valid.
- Then proceeds to step 3 (Set password) of create flow.

### Unlock (`/unlock`)
- "Welcome back."
- Password input.
- "Unlock" button.
- Below: "Wrong password? [Restore from recovery phrase]" — proceeds to restore flow.
- Below that: "Wipe this wallet" — opens danger modal.
- After 5 wrong attempts, exponential backoff (1s, 5s, 30s, 5min, 1hr).

### Dashboard (`/dashboard`)

Top bar:
- Logo · current network indicator (green dot for mainnet, yellow for testnet) · lock icon (manual lock) · settings cog.

Hero (one card):
```
┌──────────────────────────────────────────────────┐
│ Total: $1,234.56                                 │
│                                                  │
│  PRL    100.000 PRL  ≈ $620.00                   │
│  WPRL   100.000 WPRL ≈ $614.56                   │
│                                                  │
│  [ Send ]  [ Receive ]  [ Bridge ]               │
└──────────────────────────────────────────────────┘
```

Below: Recent activity (last 5 txs, link to History).
Below: Health banner if any RPC degraded (yellow), or "All systems normal" hidden.

### Send PRL (`/send/prl`)
1. Destination input. Validation: bech32m + HRP `prl`. Show parsed address fragment + identicon below input.
2. Amount input. Two modes: PRL or USD (toggle). Max button.
3. Fee tier: Low / Normal / High (radio). Show estimated time and fee in PRL + USD for each.
4. Preview card: "You're sending **X PRL** to `prl1p...abc` with fee **Y PRL** (~Z USD). Total: **X+Y PRL**. This cannot be undone."
5. Password input (re-auth).
6. "Send" button — must hold-to-confirm (1 second press).
7. Broadcast — show pending state with tx hash + link to explorer.
8. Confirmed (after N confirmations per Pearl).

### Send WPRL (`/send/wprl`)
Same flow but:
- Address validation = Ethereum checksum address.
- Fee tier shown as gas in gwei + ETH cost + USD.
- Show ETH balance + warn if < estimated gas.

### Receive (`/receive`)
- Tab toggle: PRL / WPRL.
- Address shown large, monospace.
- QR code (300x300).
- "Copy" button. "Show QR" / "Hide QR" toggle.
- For PRL (when HD enabled in v1.5): "Generate new address" button.

### Bridge (`/bridge`)

Direction toggle:
```
[ PRL → WPRL ]  [ WPRL → PRL ]
```

For PRL → WPRL:
1. Amount input (PRL, with USD secondary). Max button.
2. Live preview:
   ```
   You send:        10.00 PRL
   Bridge fee:      0.05 PRL (0.5%)
   You'll receive:  9.95 WPRL on Ethereum
   Estimated time:  6 Pearl confirmations + 1 Eth confirmation ≈ 8 minutes
   ```
3. "Recipient" — default is the user's own Eth address from this wallet. Advanced toggle to send to a different address (with extra-loud warning).
4. Preview card: "Bridging 10 PRL → 9.95 WPRL to 0x...your-eth-address. This cannot be undone." + irreversibility note.
5. Password to authorize signing.
6. "Bridge" — hold to confirm.
7. Status screen:
   - Step 1: Pearl deposit broadcast — pending → confirmed (live confirmation counter)
   - Step 2: Relayer signature — pending → received
   - Step 3: Eth mint — pending → confirmed
   - Done.

For WPRL → PRL:
1. Amount (WPRL). Max button.
2. Preview shows burn fee + Eth gas + expected PRL received.
3. Recipient defaults to user's own Pearl address; advanced to override.
4. Password.
5. Sign + broadcast WPRL burn tx.
6. Status:
   - Step 1: Eth burn broadcast — pending → confirmed
   - Step 2: Relayer signature — pending → received
   - Step 3: Pearl release — pending → confirmed
   - Done.

### Bridge status (`/bridge/status/:id`)
- Live updating progress (poll relayer + RPC every 10s).
- Cancel button is GREYED OUT with copy: "Bridge actions can't be cancelled once broadcast."
- Tx hashes for each leg with explorer links.
- Estimated time remaining.
- On failure (rare): clear error + "Contact: see docs/help — never send your mnemonic to support."

### History (`/history`)
- List of last 50 txs, all chains.
- Filter: All / PRL / WPRL / Bridge.
- Each row: direction icon (send/recv/bridge), amount, counterparty (truncated address or label), timestamp, status.
- Click → tx detail with hash + explorer link + raw tx (collapsible).

### Settings (`/settings`)
- **Account**
  - Lock now (button)
  - Auto-lock timeout (1/5/15/60 min/never)
  - Change password
- **Security**
  - Export recovery phrase (requires password re-entry)
  - View public addresses
  - Wipe wallet from this browser (danger zone, double-confirm + typed phrase)
- **Network**
  - Mainnet / Testnet toggle
  - Pearl RPC endpoint (read-only display, "managed by pearlwallet.xyz")
  - Eth RPC endpoint (read-only display)
- **Display**
  - Dark mode toggle
  - Currency display (USD only in v1)
- **About**
  - Version, build hash, audit report link, GitHub link

### About / How this works (`/about`)
- Plain-language explanation of:
  - What "non-custodial" means.
  - Why we can't recover your wallet for you.
  - What a recovery phrase is and how to store it.
  - What the bridge actually does (lock-and-mint).
  - What we DON'T do: spend your funds, see your balance, hold your keys.
  - How to verify the bundle hash matches the audited release.
- Version + build hash.
- GitHub link.
- Most-recent audit report (PDF link).

## Error catalog (sample — full version in `src/ui/copy/errors.ts`)

| Code | User-facing | Trigger |
|------|-------------|---------|
| `E_RPC_DOWN` | "Network is unreachable. Try again in a moment." | Pearl or Eth RPC failed 3x with backoff |
| `E_INSUFFICIENT_FUNDS` | "Not enough PRL to send this amount + the fee." | Coin selection failed |
| `E_INSUFFICIENT_GAS` | "Not enough ETH for gas. Add ETH or pick a lower gas tier." | Eth balance < estimated gas |
| `E_INVALID_ADDRESS` | "That doesn't look like a valid Pearl address. It should start with `prl1p`." | bech32m decode failed |
| `E_INVALID_ADDRESS_ETH` | "That doesn't look like a valid Ethereum address." | Checksum failed |
| `E_PASSWORD_WRONG` | "Incorrect password." (no other info) | PBKDF2 decrypt failed |
| `E_PASSWORD_BACKOFF` | "Too many wrong attempts. Try again in {N} seconds." | Failed-attempt counter |
| `E_BRIDGE_RELAYER_DOWN` | "Bridge service is temporarily unavailable. Your funds are safe; please try again later." | Relayer 5xx or timeout |
| `E_BRIDGE_DAILY_LIMIT` | "Today's bridge limit has been reached. Please try again tomorrow." | Contract daily limit hit |
| `E_BUNDLE_HASH_MISMATCH` | "This page may have been tampered with. Don't enter your password. Verify at github.com/.../releases." | SRI failure or build-info check failed |

## Accessibility

- **WCAG AA target**. Target ratios: ≥4.5:1 for body, ≥3:1 for large text and UI.
- All inputs have visible labels (no placeholder-only labels).
- All interactive elements keyboard-reachable in logical order.
- Screen-reader labels on QR codes ("Pearl receive address QR code, value: prl1p...").
- Reduced-motion preference respected (no animation on `prefers-reduced-motion: reduce`).
- Mnemonic display is screen-reader compatible (each word in its own `<li>` with index).

## Copy voice

- Plain English. 8th-grade reading level (Hemingway grade ≤ 8).
- No marketing fluff. No "amazing", "secure" (security is shown, not claimed), no "easy".
- Address user as "you", not "the user".
- Cap each sentence at 20 words where possible.
- Errors are descriptive of cause + action, not blamey.
